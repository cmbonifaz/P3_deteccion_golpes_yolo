Propuesta de Proyecto: Sistema Inteligente para la Detección Automática de Daños en Vehículos mediante YOLO 
El proyecto consiste en desarrollar un sistema de visión por computadora basado en el modelo YOLO (You Only Look Once) capaz de detectar automáticamente daños visibles en vehículos a partir de fotografías. La aplicación permitirá al usuario cargar una imagen de un automóvil y, mediante un modelo previamente entrenado con un conjunto de imágenes etiquetadas, identificará distintos tipos de daños como rayones, abolladuras, faros rotos, parachoques dañados, parabrisas agrietados o espejos rotos. Una vez procesada la imagen, el sistema mostrará los daños detectados mediante cuadros delimitadores sobre la fotografía y extraerá los datos estructurados de cada detección (tipo, cantidad y ubicación aproximada), los cuales serán posteriormente procesados por una capa de inteligencia artificial explicativa para generar el reporte final y la evaluación general del estado del vehículo, como se detalla más adelante.

Para el desarrollo del proyecto se utilizará como base un conjunto de datasets públicos de imágenes de vehículos con diferentes tipos de daños ya etiquetados. En caso de que se requiera cubrir clases de daño no representadas o balancear la cantidad de ejemplos por clase, se complementará el conjunto de entrenamiento mediante etiquetado manual adicional con herramientas como Roboflow o LabelImg. Posteriormente, el modelo será integrado en una aplicación desarrollada en Python que utilizará OpenCV para el procesamiento de imágenes y una interfaz sencilla que permita al usuario cargar fotografías y visualizar los resultados obtenidos.

Para la generación del estado general del vehículo y su respectivo reporte, el sistema incorpora una capa de inteligencia artificial explicativa basada en un modelo de lenguaje grande (LLM), específicamente Gemini Flash-Lite de Google. Esta capa recibe como entrada los datos estructurados obtenidos por el modelo YOLO (tipo de daño, nivel de confianza y ubicación aproximada de cada detección) y, mediante un prompt diseñado para restringir sus respuestas exclusivamente a la evidencia detectada, genera una evaluación del estado general del vehículo junto con una justificación textual comprensible para el usuario. De esta manera, el sistema no solo informa qué daños existen, sino que explica el razonamiento detrás de su conclusión, incrementando la transparencia y confiabilidad del proceso frente a un enfoque de puntuación puramente numérica.
Datasets
Automobile Damage Detection Dataset (dataset base) — cerca de 6,900 imágenes anotadas con 14 categorías de daño, incluyendo daño de parabrisas delantero/trasero, faro, luz trasera, espejo lateral, y abolladuras en cofre, cajuela, puerta, guardafango, parachoques delantero/trasero y techo. Existe un modelo ya entrenado (YOLOv11n) sobre este dataset como referencia de qué mAP esperar.
Car Damage Detection (CAPSTONE) — dataset complementario de 3,226 imágenes anotadas enfocado en componentes críticos como parachoques, puertas, guardafangos y parabrisas, útil para reforzar el volumen de ejemplos en esas clases.
Car Damage Detection (Oleksy1121) — dataset complementario necesario para cubrir la clase de rayones, ausente en el dataset base; incluye también óxido, decoloración de pintura, grietas de pintura, abolladuras y "PDR dents", con 456 imágenes base y 1,140 con data augmentation, anotadas con polígonos. Este repositorio sirve además como referencia de implementación, ya que incluye entrenamiento en Colab, comparación YOLOv8 vs Faster R-CNN, y una app en Python con GUI para visualizar resultados.
